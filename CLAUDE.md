## プロダクト概要（1行）
週ごとの科目ノルマから、移動時間・営業時間・個人予定を考慮した1日の勉強スケジュールを自動生成し、**当日の朝に「開封」するまで場所を秘密にするデイリーガチャ形式**の自分用アプリ。将来のマルチユーザー公開も見据える。

## 技術スタック（確定・変更禁止）
- Next.js (App Router) + TypeScript
- DB: Postgres (Neon) + Prisma
- 認証: Auth.js (NextAuth) の Googleログイン（Calendar連携とOAuth兼用）
- 外部API: Google Routes API（移動時間 transit+walking）/ Places API（営業時間）/ Calendar API / Open-Meteo API（天気予報・無料・キー不要）
- グラフ: Recharts / テスト: Vitest / ホスティング: Vercel

## 絶対に守るルール
1. **Google API結果は必ずDBにキャッシュ**（有効期限30日）。キャッシュなしのループ呼び出し厳禁。
   天気(Open-Meteo)は性質が異なり**同日内は再取得しない**方式でDayStateにキャッシュする（`src/lib/weather/`）。
2. **全テーブルに userId を持たせ、全クエリを userId でスコープ**する（現在は1ユーザーでも）。
3. APIキー・OAuthシークレットはサーバー側の環境変数のみ。クライアント露出・コミット禁止。
4. スケジューリングロジックは `src/lib/scheduler/` に**純粋関数**として分離し、Vitestでテストする。
5. UIはスマホ優先のレスポンシブ。
6. 外部API失敗時のフォールバック（手動入力UI）を用意する。

## ドメインルール（スケジューラの核心）
- **毎日が勉強日**（ターゲットは休学中・長期休暇の人）。休息日機能は作らない。週は月曜開始。1日の開始は出発地点・起床時刻（設定値）。**出発後21:00まで帰宅しない**。移動は常に直前の場所を起点に計算。帰宅の移動時間は考慮しない。
- 出発地点（`StartPoint`）は複数登録できる（自宅・実家等）。1ユーザー1件がデフォルト。**選択の単位は日**: 開封前の今日のページで日ごとに変更できる（開封後は変更不可）。週間生成時点ではデフォルト出発地点のみを使う。
- 出発地点は勉強場所ではない。21:00以降に勉強ブロックを置かない。
- 勉強ブロックは1場所につき最小60分。同じ場所は1日1回（連続滞在中の休憩・昼食はOK）。
- cafe種別は最大滞在時間あり（休憩込み合計）。場所は営業時間内のみ使用。
- 今週の抽選対象の場所は`WeeklyPlan.locationPoolJson`で絞り込める（週はじめ設定画面でチェックボックス選択、最低2箇所必須、未設定なら全場所）。
- `morning` タグ科目（数学・アルゴリズム系）は午前枠（〜12:00）に優先配置。
- 場所は直近数日使ったものの優先度を下げてローテーション（場所を変える楽しさ）。
- 休憩・昼食は自動で適度に配置（極端な連続勉強・昼食抜きは避ける）。
- ノルマ合計 > 実質可能時間なら生成時に警告し、優先順位の高い科目から割り当て。自動でノルマは変えない。週の途中から設定した場合も、残り日数ベースで自動的に判定される（`src/lib/scheduler/dayRange.ts`）。
- 天気は開封時にのみ取得するため、週間プラン生成時点では雨の日判定は使わない。**リロール時のみ**、開封時に取得済みの当日天気から「日中(出発〜21:00)の最大降水確率50%以上」を雨の日と判定し、その日の出発地点からの移動時間が近い場所を優先する重み付け抽選（近い半分の場所の当選確率2倍、完全排除はしない）を行う。
- **週の途中からのプラン生成に対応**: `weekly-plan`で設定した当日〜その週の日曜だけを生成対象にできる（過去の曜日は触らない）。AIノルマ提案・ノルマ超過警告・ダッシュボードの理想ペース線もすべて実際の開始日（`WeeklyPlan.startDate`）を基準に計算する（月曜起点固定にしない）。

## デイリーガチャ（表示制御の核心）
- 生成した週間プランは事前に見せない。**当日にユーザーが開封して初めてその日の場所・時間割を表示**。
- 逃げ道は2つだけ：**リロール1日1回**（当日の場所だけ引き直し、ノルマ配分は維持）と**ギブアップ**（当日分を通常表示＋手動編集解放）。
- 進捗ダッシュボード（ノルマ達成率）は常時見せてよい。隠すのは未開封日の場所・時間割のみ。
- Googleカレンダーへは場所・科目を伏せたマスクイベント（時間帯のみ正しい）で書き出し、開封時に当日分を実名に更新。
- 週の準備を促す仕組み: 毎週日曜18:00にVercel Cron（`vercel.json` / `src/app/api/cron/weekly-prep-reminder/`）がGoogleカレンダーへリマインダーを自動作成。日曜18時〜月曜は今日のページにバナー表示。今週のプラン自体が未生成なら「準備中」表示でガチャは引けない。
  - **要設定**: VercelプロジェクトのCRON_SECRET環境変数（cronルートの認証用、この開発環境からは設定不可）。

## ディレクトリ方針
- `src/lib/scheduler/` 純粋ロジック（API・DB非依存）。`dayRange.ts`が週間生成・週途中再生成の共通コア
- `src/lib/google/` Google API クライアント＋キャッシュ層
- `src/lib/weather/` Open-Meteo クライアント＋当日天気キャッシュ＋3時間集約ロジック（雨の日判定・場所抽選の重み付けに使用）
- `src/lib/ai/` Claude API クライアント＋AI関連ロジック（週次ノルマ提案・参考書vision抽出・ミッション文生成）
- `src/app/start-points/` 出発地点CRUD（`src/app/locations/`と同構成）
- `src/app/api/cron/` Vercel Cronから呼ばれるルート（セッション認証は通さず、CRON_SECRETヘッダーで独自認証）
- `src/app/` 画面とAPI Routes / `prisma/` スキーマ / `tasks/` フェーズ別作業指示
