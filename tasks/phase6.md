# Phase 6: AI活用（ノルマ提案・ミッション文）・参考書・ストリーク

## ゴール
AIが計画サイクルの中心に入り（週次ノルマ提案）、参考書進捗とストリークで継続動機が強化された状態。ハッカソン提出版の目玉機能。

## 前提
- `ANTHROPIC_API_KEY` をサーバー側環境変数に追加（`.env.example`にも追記）
- AI呼び出しはすべてServer Action / サーバー側から。クライアントにキーを露出させない
- モデルは claude-sonnet-4-6 等の標準モデルでよい。応答はJSONで受け取りパースする
- **既存のスケジューラ（generateWeek等）には一切手を入れない**。AIは「計画の入力（ノルマ・タグ）への提案」と「文章生成」のみを担い、スケジュール計算はアルゴリズムのまま

## 6-1. 週次AIノルマ提案（案X・一括承認型）

### データ収集
`buildWeeklyReviewInput(userId)`: 直近数週間の実績を集計して返す純粋関数
- 科目別: ノルマ・実績分・達成率・時間帯別の未実施率（morning枠の未実施率など）
- 参考書別: 進捗率・実測ペース（分/ページ、ReadingLogがあれば）
- 行動: リロール使用回数、ギブアップ回数、ストリーク状況
- データ量の指標: 実績のある週数・ブロック数（信頼度表示に使う）

### AI提案の生成
`generateWeeklyProposal(input)`: 上記データをClaude APIに渡し、JSONで受け取る
- 出力: 科目ごとの提案ノルマ（分）・timeSlot変更案・優先順位案・理由文（日本語1〜2文/科目）・全体コメント（先週の振り返り）
- **信頼度（AtCoder初期レート方式）**: 実績データ量に応じて confidence: "provisional" | "established" を返させる
  - provisional（実績2週未満など）: UIに「⚠️ まだ実績データが少ないため暫定的な提案です。精度は使うほど上がります」と注意書きを表示
  - established: 注意書きなし
- プロンプトには「ノルマを勝手に増やしすぎない」「未達が続く科目は減らす方向を優先」「変更がない科目は変更なしと明示」等の方針を含める

### UI（週はじめ優先順位設定画面を拡張）
- ページを開くとAI提案カードを表示: 全体コメント → 科目別の提案（現行→提案、理由付き）→ 信頼度注意書き（provisional時）
- ボタンは2つだけ（**一括承認型**）:
  - 「この提案で生成」→ 提案値をSubject/WeeklyPlanに反映してプラン生成
  - 「自分で決める」→ 従来の手動設定画面へ（AI提案は参考表示として残す）
- AI呼び出し失敗時は従来の手動画面に自然にフォールバック（エラーで週初め設定が止まらないこと）

## 6-2. 参考書（Book）機能

### スキーマ追加
- Book(id, userId, subjectId, title, publisher?, totalPages?, coverImageUrl?, completedAt?)
- ReadingLog(id, userId, bookId, scheduleBlockId?, fromPage, toPage, date)

### AI表紙登録
- 科目管理画面（または専用画面）に「📷 参考書を追加」
- 画像アップロード → サーバーでbase64化 → Claude API（vision）に登録済み科目リストと共に送信
- 返却JSON: title / publisher / 推定totalPages（不明ならnull）/ 最も近いsubjectId
- **フォームの自動入力（下書き）として表示し、ユーザーが確認・修正して保存**。AI出力を無確認で保存しない。totalPagesは「実物で確認してください」の注意書き付き
- 画像なしの手動登録も可能にする（AIはあくまで入力補助）

### ページ実績記録
- 今日のページの勉強ブロックに追加（任意入力）:
  - 使った参考書（そのブロックの科目に紐づくBookのみのプルダウン）
  - 進んだページ（from〜to）
- 未入力でも完了記録・ストリークに影響しない

### 進捗表示・制覇演出
- ダッシュボードに参考書ごとの進捗バー（現在ページ/totalPages、%）
- 進捗100%到達時に「📕 1冊制覇！」演出 → completedAtを記録
- 制覇済みの本が並ぶ「本棚」セクション（表紙画像 or タイトルカード）

## 6-3. ストリーク（Duolingo参考・案B）

### スキーマ追加
- Streak(userId, currentCount, longestCount, lastQualifiedDate, freezeUsedInWeek: Boolean)

### ルール
- **その日の勉強ブロックを1つ以上「完了」or「一部完了」にした日 = ストリーク継続**（開封だけでは継続しない）
- 途切れる場合、**週1回まで自動フリーズ**で救済（フリーズ発動をUIで明示: 「🧊 フリーズを使ってストリークを守りました」）。フリーズは週の開始（月曜）にリセット
- 判定は日付基準（JST）。日付処理は既存のtodayDateString等と同じ基準を使うこと

### UI
- 今日のページのヘッダー付近に 🔥+日数 を常時表示
- 開封演出時に「🔥 7日目！」を一緒に表示（6-4と連動）
- ダッシュボードに最長記録を表示

## 6-4. 開封ミッション文生成（演出強化・実装最小）

- 開封時、その日のスケジュール（場所・科目・時間）＋ストリーク日数をClaude APIに渡し、2〜3文の「本日の任務」風テキストを生成して開封演出に表示
- 生成失敗時・遅延時は固定文にフォールバックして開封をブロックしない（AI応答を待って開封が遅くなるのはNG。非同期で後から差し込む形でも可）
- コスト対策: 1日1回（開封時）のみ生成。リロール時は再生成してよいが、それ以外で再呼び出ししない
