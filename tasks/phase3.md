# Phase 3: タイムラインUI・管理画面

## ゴール
スマホで使える画面一式が動き、週間プラン生成〜手動リスケまでブラウザで完結する。

## 画面（スマホ優先レスポンシブ）
1. **設定**：自宅住所、起床時刻（平日/休日）、morning枠終了時刻（デフォルト12:00）
2. **科目管理**：CRUD、週間ノルマ（分）、timeSlotタグ
3. **週はじめ優先順位設定**：その週の科目をドラッグで並べ替え（WeeklyPlan.priorities に保存）
4. **勉強場所管理**：CRUD。住所入力→placeId解決→営業時間自動取得（失敗時は手動入力フォーム表示）。種別・最大滞在時間
5. **個人予定入力**：CRUD、場所の紐づけ（任意）
6. **今日のページ（メイン画面・デイリーガチャ）**：
   - 週はじめ画面の「今週のプランを生成」で内部生成（**結果は見せない**。ノルマ超過警告のみ表示）
   - 未開封：開封ボタン＋簡単な開封演出（カードをめくる程度でよい）
   - 開封後：当日のタイムライン表示（移動/勉強/休憩/昼食/予定を色分け）
   - **リロールボタン（残1回表示）**：Phase 1 の reroll を呼ぶ
   - **ギブアップボタン**：当日分の手動編集（時刻・場所変更・削除。編集モーダルで可）を解放
   - 「今日以降を再計画」ボタン（Phase 1 の reschedule。再生成された未来日は未開封に戻す）
   - **未開封日の場所・時間割はAPIレスポンスにも含めない**（クライアントに送ったら開発者ツールでネタバレするため。開封APIを叩いて初めて返す）

## 実装方針
- データ取得・更新は Server Actions または API Routes（userId スコープ必須）
- この段階では認証は仮（固定ユーザー）でよい。Phase 4 で差し替え。

## 完了条件
- ダミーではなく実データ（自分の科目・場所・予定）で1週間分のプランが画面に出る
- CLAUDE.md の Phase 3 にチェック
