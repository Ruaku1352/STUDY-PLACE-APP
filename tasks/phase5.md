# Phase 5: 実績記録・進捗グラフ・デプロイ

## ゴール
実績を記録して進捗が可視化され、Vercelで公開されて実運用（自分の1週間）が始められる。

## やること
1. **実績入力**：週間プランページの各勉強ブロックに「完了 / 一部完了（分数入力）/ 未実施」ボタン。ScheduleBlock.status / actualMin に保存。スマホで片手操作できるUIにする。
2. **進捗ダッシュボード**（Recharts）：
   - 科目別：今週の実績分 / ノルマ分 の達成率バー
   - 理想ペース線：週の経過割合（例:水曜終了時点=3/7）と各科目の達成率を比較し、遅れている科目を色で強調
   - （余裕があれば）過去数週間の達成率推移の折れ線
3. 実績が reschedule の残ノルマ計算に反映されることを確認（Phase 1 との結合テスト）
4. **Vercelデプロイ**：
   - Neon本番DB接続、環境変数設定（Google各キー、ALLOWED_EMAILS）
   - Google OAuth のリダイレクトURIに本番URLを追加
5. 実運用テスト：本物の自分の科目・場所・予定で1週間分を生成→カレンダー同期→通知確認、まで通す

## 完了条件
- スマホから実績入力→ダッシュボードに反映、まで本番URLで動く
